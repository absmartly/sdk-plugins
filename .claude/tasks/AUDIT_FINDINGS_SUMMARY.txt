================================================================================
KEY ERROR HANDLING PATTERNS
================================================================================

Pattern 1: DEBUG-ONLY LOGGING
  Problem: Errors only logged when debug=true
  Example: DOMManipulatorLite.ts line 266
  Impact: Production errors invisible without debug mode enabled

Pattern 2: SILENT FAILURES
  Problem: Operations fail but return false without context
  Example: DOMChangesPluginLite.ts line 946
  Impact: Users don't know what failed or why

Pattern 3: OVERLY BROAD CATCH BLOCKS  
  Problem: Catches 20+ line sections, same error handling for all
  Example: DOMManipulatorLite.ts lines 56-275
  Impact: Can't distinguish selector errors from permissions errors

Pattern 4: UNHANDLED PROMISES
  Problem: Async functions not awaited, no .catch() handlers
  Example: refreshExperiments() at line 1028
  Impact: Unhandled promise warnings + silent failures

Pattern 5: SILENT API DEGRADATION
  Problem: API/SDK failures don't inform user
  Example: OverridesPlugin fetch failures
  Impact: Users don't know critical features are unavailable

================================================================================
RECOMMENDED IMMEDIATE ACTIONS
================================================================================

1. Add error event emission for critical operations:
   - DOM change application failures
   - API/SDK fetch failures  
   - Exposure tracking failures

2. Distinguish error types:
   - Expected failures (element not found) - log as debug
   - Unexpected failures (permissions, parsing) - log as error

3. Handle all unhandled promises:
   - Add .catch() to refreshExperiments()
   - Add .catch() to trackWebVitalsMetrics()
   - Add try/await in event listeners

4. Add specific error logging:
   - Include selector in DOM errors
   - Include URL in API errors
   - Include code snippet in JS errors

5. Document error handling expectations:
   - What errors should be expected
   - How users should handle errors
   - Where to find error information
